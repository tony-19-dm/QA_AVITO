# Баг-репорт API микросервиса объявлений

## Критические баги

### BUG-001: Несоответствие формата ответа при создании объявления
**Описание**: При успешном создании объявления сервер возвращает ответ в формате, отличном от ожидаемого по спецификации Postman.

**Ожидаемый результат** (согласно Postman):
```json
{
  "id": "<string>",
  "sellerId": "<integer>",
  "name": "<string>",
  "price": "<integer>",
  "statistics": {
    "likes": "<integer>",
    "viewCount": "<integer>",
    "contacts": "<integer>"
  },
  "createdAt": "<string>"
}
```

**Фактический результат**:
```json
{
  "status": "Сохранили объявление - eb34abff-1034-4796-aa21-84ad94e051a8"
}
```

**Влияние**: Клиенты не могут получить ID созданного объявления для последующих операций.

### BUG-002: Некорректные коды статусов для несуществующих ресурсов
**Описание**: При запросе несуществующих объявлений сервер возвращает 400 вместо 404.

**Endpoint**: `GET /api/1/item/:id`, `GET /api/1/statistic/:id`, `DELETE /api/2/item/:id`
**Ожидаемый результат**: 404 Not Found
**Фактический результат**: 400 Bad Request

**Влияние**: Клиенты не могут отличить ошибку валидации от отсутствия ресурса.

### BUG-003: Отсутствие валидации минимальных значений
**Описание**: Сервер не принимает объявления с минимальными значениями (цена=1, статистика=0)

**Endpoint**: `POST /api/1/item`
**Ожидаемый результат**: 200 OK
**Фактический результат**: 400 Bad Request

**Влияние**: Ограничивает функциональность создания объявлений с низкой ценой.

## Баги средней важности

### BUG-004: Неподдержка специальных символов в названии
**Описание**: Сервер возвращает ошибку при создании объявления со специальными символами в названии.

**Влияние**: Ограничивает интернационализацию и использование не-ASCII символов.

### BUG-005: Несоответствие формата статистики
**Описание**: Формат ответа статистики не соответствует спецификации.

**Ожидаемый результат**: Массив объектов статистики
**Фактический результат**: Непредсказуемый формат

## Предложения по исправлению

1. **Стандартизировать форматы ответов** согласно документации Postman
2. **Исправить коды статусов** для несуществующих ресурсов
3. **Добавить валидацию** с четкими сообщениями об ошибках
4. **Поддержать Unicode** в текстовых полях
